## 1. 프로젝트 개요
본 프로젝트는 SNS 상의 키워드 흐름을 정량화하여 트렌드의 확산·소비 패턴을 분석하고,  
이를 기반으로 **트렌드 파워 지수(TPI)** 를 구축해 기업이 상품 기획·발주·마케팅 의사결정을  
보다 빠르고 정확하게 수행할 수 있도록 하는 것이 목표입니다.

특히 시장 경쟁이 치열하고 트렌드 변화 속도가 빠른 **편의점 산업(BGF리테일)** 을 타겟으로 하여,  
“지역·연령대에 따라 다르게 나타나는 트렌드를 반영한 발주 및 상품 기획”이라는  
실질적인 문제를 해결하고자 했습니다.

---

## 2. 문제 정의

### 2-1. 기존 트렌드 분석의 한계
- SNS 데이터를 개별 플랫폼 단위로만 수집 → 종합적인 트렌드 판단 불가  
- 긍/부정, 댓글·좋아요, 조회수 등 정성·정량 지표를 통합한 분석 미흡  
- 동일한 기준의 발주 → 지역·연령대별 수요 차이를 반영하기 어려움  

### 2-2. 기업 관점에서의 Pain Point
- MZ세대를 중심으로 신규 트렌드의 변화 주기가 매우 짧아짐  
- 트렌드 연계 상품의 성공 여부가 매출에 큰 영향  
- 편의점 업계는 PB 경쟁 심화 → 선제적 트렌드 대응 필요  

➡ **트렌드를 빠르게 감지·예측하고 이를 상품 전략에 연결하는 통합 지표가 필요했습니다.**

## 3. 시장 조사

### 3-1. 시장 흐름 분석
- 트렌드는 실제 소비시장(편의점 매출 등)에 직접적인 영향  
- CU는 2025 핵심전략으로 ‘SMOOTH: 부드러운 변화 대응’을 제시  
  → 트렌드 기반 선제 대응 강조  

### 3-2. 편의점 업계 트렌드
- PB 브랜딩 경쟁 심화  
- SNS 흐름이 제품(음료, 간식, 건강식, 라이프스타일 품목 등) 판매에 직접 연결  

### 결론  
SNS는 소비시장 움직임의 선행지표이며,  
이를 정량화할 수 있는 **트렌드 지표가 필요하다.**

## 5. 트렌드 파워 지수(TPI) 구축

### 사용 데이터
- 커뮤니티  
- 인스타그램  
- 블로그  
- X(트위터)  
- 유튜브  

(조회수, 긍정, 부정, 중립 등 반응 변수 활용)

### 5-1. 스케일링
각 변수의 단위 차이를 제거하기 위해 모든 값을 스케일링 적용.

### 5-2. Isolation Forest 기반 anomaly score 산출
트렌드 급등 구간을 탐지하기 위해 Isolation Forest 활용.

#### 사용 이유
- 트렌드 데이터는 특정 시점에 **갑자기 폭발적으로 증가하는 패턴**이 많음  
- Isolation Forest는  
  → **비정상적으로 높거나 급변한 데이터포인트(=잠재적 트렌드 발생 지점)** 을  
    효과적으로 탐지할 수 있음  

### 5-3. anomaly score → TPI 산출
- 각 데이터포인트에 대해 anomaly score 산출  
- anomaly score가 높을수록 트렌드 변화 지점으로 판단  
- anomaly score를 **0~100 범위로 스케일링**하여 최종 **TPI 값 도출**

## 6. 기업 적용 솔루션 제안

### 6-1. 발주 최적화 시스템
- 지역 기반 트렌드 + 기존 판매량 데이터를 결합  
- 높은 트렌드 지수를 보이는 키워드 관련 제품 우선 발주  
- 재고 효율 개선, 폐기 비용 감소 효과  

### 6-2. 제품 추천 서비스
- 특정 키워드와 연관된 제품 자동 추천  
- 경쟁사 대비 빠른 상품 기획·출시 가능

## 7. SNS 트렌드 분석의 한계점 및 개선 방향

### 7-1. 지역별 수요 반영의 한계

#### 한계점
현재 TPI 산출 로직에서는 **지역별 트렌드 반응도 차이**가 반영되지 않았다.  
모든 지역이 동일한 트렌드 강도로 반응한다는 가정 하에 TPI가 계산되었으며,  
이는 실제 소비 행태와 괴리가 발생할 수 있다.

- 동일한 트렌드 키워드라도  
  → 지역별 연령 구성, 소비 성향, 구매력에 따라 **판매 전환율이 상이**
- 코드 상에서도 **지역별 가중치(Weight)**가 적용되지 않음

#### 개선 방향
과거 트렌드 상품의 **지역별 판매량 데이터**를 활용하여  
각 지역의 **트렌드 민감도** 를 추정한 뒤  
이를 TPI 산출 시 **가중치로 반영하는 구조**가 보다 합리적일 것이라고 생각된다.

### 7-2. Isolation Forest 기반 이상 탐지 모델의 한계

#### 한계점
본 프로젝트에서는 트렌드 급등 구간 탐지를 위해  
**Isolation Forest(이상치 탐지 모델)** 을 사용하였다.

그러나 Isolation Forest의 특성상,

- 값이 **급격히 상승한 경우**
- 값이 **급격히 하락한 경우**

모두 **동일한 이상치 수준**으로 판단될 수 있음

→ 이는  
- **트렌드 폭발(긍정적 신호)**  
- **관심 급감(부정적 신호)**  

를 구분하지 못해 **해석 상 혼동을 유발할 가능성**이 있다.

#### 개선 방향
트렌드의 **방향성** 을 함께 고려할 수 있는  
모델 또는 지표 도입이 필요하다.
